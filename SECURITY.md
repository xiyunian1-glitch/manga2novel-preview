# Security Policy

## Reporting a Vulnerability

Please do not disclose security vulnerabilities in public GitHub issues.

If you find a vulnerability, contact the maintainer through a private channel if one is available. If no private contact channel is available, open a GitHub issue only to request a private reporting path and do not include exploit details, secrets, or reproduction steps in the public report.

## Supported Versions

Security fixes are provided on a best-effort basis for the latest code on the `main` branch.
